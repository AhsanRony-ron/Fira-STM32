# System Files README

## STM32 System Files

This folder contains STM32CubeMX-generated system initialization and configuration files.

### Files

- **system_stm32f4xx.c** - System clock configuration
  - PLL configuration
  - Clock tree setup
  - FLASH latency settings

- **stm32f4xx_it.c** - Interrupt handlers
  - EXTI handlers (external interrupts)
  - UART/SPI/Timer ISRs
  - DMA interrupts

- **stm32f4xx_hal_msp.c** - HAL board support package
  - GPIO initialization for hardware resources
  - Timer initialization
  - Peripheral clock enables

## Integration Notes

These files are typically generated by **STM32CubeMX** and should be regenerated if:
- You add/remove peripherals in CubeMX
- You change clock configuration
- You add new interrupts or DMA channels

### CubeMX Regeneration

1. Open your .ioc file in STM32CubeMX
2. Make changes to device configuration
3. **Project → Generate Code**
4. Copy generated files from Drivers/CMSIS and Drivers/STM32F4xx_HAL_Driver
5. Update system_stm32f4xx.c and stm32f4xx_it.c in this folder

## Custom Modifications

If you need to add custom code to these files when regenerating:

### stm32f4xx_it.c
Add custom interrupt handlers in the marked sections (USER CODE BEGIN/END):

```c
void EXTI0_IRQHandler(void)
{
    /* USER CODE BEGIN EXTI0_IRQn 0 */
    // Your code here (e.g., sensor interrupt)
    /* USER CODE END EXTI0_IRQn 0 */
    
    HAL_EXTI_IRQHandler(GPIO_PIN_0);
}
```

### stm32f4xx_hal_msp.c
Add GPIO/peripheral init in relevant *_MspInit functions:

```c
void HAL_TIM_PWM_MspInit(TIM_HandleTypeDef *htim)
{
    /* USER CODE BEGIN TIM2_MspInit 0 */
    // Your custom initialization
    /* USER CODE END TIM2_MspInit 0 */
}
```

## Do Not Modify

- Clock dividers without understanding impact
- Interrupt priorities carelessly (deadlocks possible)
- HAL_Init() or MX_XX_Init() function signatures

## Migration from src/Additional/

If using old code from `src/Aditional/`:

1. Compare system_stm32f4xx.c with old version
2. Merge any custom clock configurations
3. Compare stm32f4xx_it.c and merge interrupt handlers
4. Verify all changes in source control before building
